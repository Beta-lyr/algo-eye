### 架构优势（已验证）

- **三段式管线**：算法生成器 → 引擎回放 → 渲染器绘制，解耦彻底

- **渲染器可复用**：`ArrayRenderer` 覆盖所有排序和搜索，`GridRenderer` 覆盖图算法

## 3. 方向二：交互增强（从"看"到"玩"）

### 3.1 算法对比模式

**需求**：同时运行两个算法，直观比较效率差异

**实现方案**：
- 画布左右分屏，各跑一个算法
- 共享同一份输入数据
- 同步播放，实时显示步数/比较次数/交换次数
- 胜负判定：谁先完成、谁的步数更少

```ts
// 新增 store 状态
interface CompareState {
  enabled: boolean;
  leftAlgo: Algorithm | null;
  rightAlgo: Algorithm | null;
  synced: boolean;  // 是否同步播放
}
```

**UI 变化**：
- 顶栏增加"对比模式"开关
- 画布区域分为 `left-canvas` + `right-canvas`
- 统计面板左右对称显示

### 3.2 手动操作模式

**需求**：允许用户手动执行每一步（拖拽交换、点击比较）

**实现方案**：
- 增加"手动模式"按钮
- 在手动模式下，用户点击两个元素触发比较/交换
- 验证操作是否合法（如冒泡排序只能交换相邻元素）
- 提供提示：哪些操作是合法的

### 3.3 步骤书签与注释

**需求**：标记关键步骤，添加学习笔记

**实现方案**：
- 步骤进度条上可点击添加书签
- 书签支持简短注释（终端风格输入框）
- 书签列表面板，点击跳转
- 导出为 JSON 分享

## 5. 方向四：渲染能力升级

### 5.1 WebGL 渲染器（大规模数据）

**触发条件**：当元素数量 > 500 时，Canvas 2D 的 `shadowBlur` 性能不足

**实现方案**：
- 新增 `WebglRenderer` 实现 `Renderer` 接口
- 使用 WebGL 的 shader 实现辉光效果（比 `shadowBlur` 快 10x）
- 自动降级：检测性能，大 n 时切换

```ts
// 渲染器工厂
function getRenderer(dataSize: number): Renderer {
  if (dataSize > 500) return new WebglRenderer();
  return new Canvas2DRenderer();
}
```

### 5.2 3D 树渲染器

**需求**：红黑树、AVL 树等复杂数据结构的 3D 展示

**实现方案**：
- 使用 Three.js 或原生 WebGL
- 节点为 3D 球体，边为圆柱
- 旋转/缩放交互
- 保留 CRT 风格（暗底 + 绿色辉光）

### 5.3 动画插值增强

**需求**：当前状态切换是瞬时的，缺少过渡动画

**实现方案**：
- 柱状图：高度变化 0.25s 缓动
- 颜色变化：0.15s 渐变
- 位置交换：0.3s 平移动画
- 使用 `requestAnimationFrame` + 缓动函数

## 8. 技术风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 渲染器数量爆炸 | 维护成本高 | 抽象基类 + 组合模式，复用通用绘制逻辑 |
| DP 算法步骤数巨大 | 内存膨胀 | 关键帧 + 增量快照，或按需计算 |
| WebGL 兼容性 | 部分浏览器不支持 | 自动降级到 Canvas 2D |
| 3D 渲染性能 | 移动端卡顿 | 3D 仅桌面端启用，移动端降级为 2D |
| 算法讲解内容质量 | 需要专业知识 | 初期用 AI 生成 + 人工审核 |

## 9. 优先级建议

**高优先级（用户价值高、实现成本低）**：
1. 排序算法扩展（V2.1）— 复用现有架构，1 周完成
2. 搜索/字符串算法扩展（V2.2）— 同上
3. 算法对比模式（V2.6）— 差异化功能，用户感知强

**中优先级（价值高、成本中）**：
4. 数据结构扩展（V2.3）— 需要新渲染器
5. 算法讲解页面（V2.7）— 内容创作成本
6. 动态规划可视化（V2.4）— 需要新渲染器

**低优先级（锦上添花）**：
7. WebGL 渲染器（V2.5 之后）— 性能优化
8. 3D 渲染器 — 视觉炫酷但非核心
9. 成就系统 — 增加粘性但非必需

## 10. 架构调整建议

### 10.1 算法注册表重构

当前是手动导入，V2 算法数量增多后建议改为动态注册：

```ts
// 当前：手动导入
import { bubbleSort } from './sorting/bubbleSort';
// ...

// V2：自动扫描 + 懒加载
const algorithmModules = import.meta.glob('./**/*.ts', { eager: true });
export const algorithms = Object.values(algorithmModules)
  .map(m => m.default)
  .filter(Boolean);
```

### 10.2 渲染器工厂模式

```ts
// 根据数据类型自动选择渲染器
function getRenderer(kind: Snapshot['kind']): Renderer {
  switch (kind) {
    case 'array': return new ArrayRenderer();
    case 'grid': return new GridRenderer();
    case 'tree': return new TreeRenderer();
    case 'linked-list': return new LinkedListRenderer();
    case 'dp-grid': return new DPGridRenderer();
    default: throw new Error(`Unknown kind: ${kind}`);
  }
}
```

### 10.3 算法分类与标签系统

```ts
interface Algorithm {
  // ... 现有字段
  tags: string[];  // ['comparison-based', 'in-place', 'stable', 'divide-conquer']
  difficulty: 'beginner' | 'intermediate' | 'advanced';
  relatedAlgorithms: string[];  // 关联算法 ID
}
```
